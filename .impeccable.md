## Design Context

### Users
Mac users who need to understand and reclaim disk space. Not exclusively developers — the audience ranges from power users who know what `node_modules` is to non-technical users who just know their disk is full. They're typically in a "fix it" mindset: something is wrong (disk full), and they want to find the culprit and delete it confidently. Context: stressed, impatient, don't want to break things.

### Brand Personality
**Sharp, precise, powerful.** Respects the user's intelligence without being intimidating. Feels like a tool Apple would ship — no unnecessary personality, no branding for branding's sake. Every element earns its place.

Voice: concise, direct, no filler. Labels are verbs ("Move to Trash") not nouns ("Deletion"). Errors explain what to do, not what went wrong.

### Aesthetic Direction
**Native macOS utility.** Should feel like it ships with the OS — Finder-grade polish. Reference: Finder, Disk Utility, Activity Monitor. The app should be invisible as a "third-party app."

- **Theme**: Follow system appearance (light + dark). Currently dark-only; light mode is a planned addition.
- **Colors**: macOS system blue (#0a84ff) accent. Warm grays for dark mode (#1c1c1e series). File type category colors for data encoding (folder=blue, media=orange, audio=purple, archive=yellow, code=green, document=cyan).
- **Typography**: SF Pro via -apple-system. No custom fonts. Type hierarchy through weight and size, not font variation.
- **Icons**: Lucide (stroke style matches SF Symbols aesthetic). Colored by file type category.
- **Motion**: Restrained. 100-200ms transitions for feedback. One hero animation (welcome ring chart). prefers-reduced-motion respected.
- **Anti-references**: CleanMyMac (too branded), DaisyDisk (too playful), any app with gradients, glassmorphism, or neon accents.

### Design Principles

1. **Native first**: When in doubt, do what macOS does. System font, system colors, system conventions (right-click menus, ⌘ shortcuts, Trash not Delete).
2. **Data as decoration**: The proportional bars and category colors ARE the visual design. No decorative elements needed when the data itself is visually interesting.
3. **Confidence through clarity**: Users are about to delete files. Every size, percentage, and path must be unambiguous. Trash is recoverable and the UI says so.
4. **Progressive depth**: Welcome screen is simple (2 buttons). Main view adds navigation. Right-click reveals power features. Keyboard shortcuts for experts. Each layer discovered naturally.
5. **Performance is a feature**: Streaming scan, parallel scanning, view-based updates — the UI should never freeze or feel slow. Instant feedback on every action.
